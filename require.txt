

（1）主要研究内容
1.系统需求分析：通过对台州凯绿化工有限公司的实地调研，分析其在客户管理、产品管理、订单处理、库存控制等方面的需求，明确系统功能与非功能需求。
2.系统架构设计：采用前后端分离架构，后端基于SpringBoot框架构建RESTful API，前端基于Vue.js框架实现用户界面，使用MySQL作为数据存储。
3.功能模块设计：
○客户管理模块：客户信息录入、查询、分类、联系记录等；
○产品管理模块：农药产品信息管理、分类、价格调整；
○订单管理模块：订单创建、状态跟踪、历史查询；
○库存管理模块：库存预警、出入库记录、盘点功能；
○销售统计模块：销售额统计、产品销量分析、客户购买行为分析；
○AI对话模块：集成第三方AI对话API，实现智能客服与用药建议。
○AI对话集成：选择适合的AI对话接口（如OpenAI API或国内同类服务），设计对话流程，实现基于上下文的问答与推荐功能。
○系统测试与优化：对系统进行功能测试、性能测试与安全测试，确保系统稳定可靠。
（2）预期目标
1.完成一个具备完整前后端的农药销售管理系统，实现六大核心功能模块。
2.成功集成AI对话接口，实现智能客服与产品推荐功能。
3.系统具备良好的用户体验与响应速度，支持多用户并发操作。
4.撰写完整的系统设计文档、数据库设计文档、API接口文档与使用手册。
5.通过本系统提升企业对销售流程的控制能力，为同类企业提供参考范例。
三、论文（设计）的主要研究方案（拟采用的研究方法、准备工作情况及主要措施） 
（一）研究对象及数据采集 
本系统以台州凯绿化工有限公司为研究对象，通过以下途径获取数据与需求：
○企业实地访谈与业务流程观察；
○现有Excel表格与纸质记录的数字化整理；
○同类系统功能参考与竞品分析；
○国家农药产品数据库与行业标准资料。
（二）研究方法及主要措施
（1）技术选型与环境搭建
○后端技术栈：SpringBoot + MyBatis + MySQL
○前端技术栈：Vue 3 + Element + Axios
○开发工具：IntelliJ IDEA、VS Code、Navicat、Postman
○AI接口：拟选用deepseek API
（2）系统设计与实现步骤
1.数据库设计：根据业务需求设计ER图，建立客户表、产品表、订单表、库存表、对话记录表等。
2.后端开发：
○搭建SpringBoot项目，配置数据源与MyBatis-Plus；
○编写实体类、Mapper接口、Service层与Controller层；
○实现RESTful API，包括登录验证、数据增删改查、统计查询等；
○集成AI对话接口，封装HTTP请求与响应处理。
3.前端开发：
○搭建Vue项目，配置路由与状态管理；
○使用Element Plus构建UI组件，实现各功能页面；
○通过Axios调用后端接口，实现数据交互；
○设计AI对话界面，实现实时问答与历史记录展示。
4.系统集成与测试：
○前后端联调，确保接口通信正常；
○进行单元测试、集成测试与用户验收测试；
○优化系统性能与界面交互体验。
（3）AI对话模块实现方案
○选择适合的AI对话API，申请API Key并了解调用方式；
○设计对话场景：包括产品咨询、用药指导、订单查询等；
○后端封装对话请求，处理上下文记忆与敏感词过滤；
○前端实现对话界面，支持文本输入与消息展示；
○记录对话历史，便于后续分析与优化。
